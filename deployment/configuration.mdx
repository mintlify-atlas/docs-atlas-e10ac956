## Configuration Overview

The Vantix Sales System uses ASP.NET Core's configuration system with the following hierarchy (later sources override earlier ones):

1. `appsettings.json` - Base configuration
2. `appsettings.{Environment}.json` - Environment-specific settings
3. Environment variables - Secure secrets and overrides
4. Command-line arguments - Runtime overrides

## Configuration Files

### Base Configuration (appsettings.json)

The base configuration file contains settings common to all environments:

```json Vantix.Api/appsettings.json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "AllowedHosts": "*"
}
```

### Development Configuration (appsettings.Development.json)

Development-specific settings (never use in production):

```json Vantix.Api/appsettings.Development.json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning"
    }
  },
  "ConnectionStrings": {
    "ventixDb": "Server=localhost;Database=VantixV1;Trusted_Connection=True;TrustServerCertificate=True"
  },
  "Jwt": {
    "Key": "Development_Key_Only_Not_For_Production_32_Chars!"
  }
}
```

<Warning>
  **Never deploy** `appsettings.Development.json` to production servers. This file should be excluded from production deployments.
</Warning>

### Production Configuration (appsettings.Production.json)

Create a production configuration file:

```json Vantix.Api/appsettings.Production.json
{
  "Logging": {
    "LogLevel": {
      "Default": "Warning",
      "Microsoft.AspNetCore": "Warning",
      "Microsoft.EntityFrameworkCore": "Error"
    }
  },
  "AllowedHosts": "yourdomain.com,www.yourdomain.com"
}
```

<Note>
  **Do not include sensitive data** (connection strings, JWT keys) in `appsettings.Production.json`. Use environment variables instead.
</Note>

## Database Configuration

### Connection String Format

The connection string format depends on your authentication method:

<CodeGroup>
```text Windows Authentication
Server=YOUR_SERVER;Database=VantixV1;Trusted_Connection=True;TrustServerCertificate=True;Encrypt=True
```

```text SQL Server Authentication
Server=YOUR_SERVER;Database=VantixV1;User Id=YOUR_USER;Password=YOUR_PASSWORD;TrustServerCertificate=True;Encrypt=True
```

```text Azure SQL Database
Server=tcp:yourserver.database.windows.net,1433;Database=VantixV1;User Id=YOUR_USER;Password=YOUR_PASSWORD;Encrypt=True;TrustServerCertificate=False
```
</CodeGroup>

### Using Environment Variables

Set the connection string via environment variable (recommended for production):

<CodeGroup>
```bash Linux/macOS
export ConnectionStrings__ventixDb="Server=YOUR_SERVER;Database=VantixV1;User Id=YOUR_USER;Password=YOUR_PASSWORD;Encrypt=True"
```

```powershell Windows PowerShell
$env:ConnectionStrings__ventixDb="Server=YOUR_SERVER;Database=VantixV1;User Id=YOUR_USER;Password=YOUR_PASSWORD;Encrypt=True"
```

```bash Docker
docker run -e ConnectionStrings__ventixDb="Server=YOUR_SERVER;Database=VantixV1;User Id=YOUR_USER;Password=YOUR_PASSWORD;Encrypt=True" vantix-api
```
</CodeGroup>

<Note>
  **Configuration Path Separator**: Use double underscores (`__`) to represent nested configuration in environment variables.
  
  Example: `ConnectionStrings:ventixDb` becomes `ConnectionStrings__ventixDb`
</Note>

## JWT Configuration

### JWT Settings

Configure JWT authentication for your application:

```json
{
  "Jwt": {
    "Key": "YOUR_SECRET_KEY_MINIMUM_32_CHARACTERS",
    "Issuer": "VantixSalesSystem",
    "Audience": "VantixApiUsers",
    "ExpirationMinutes": 60
  }
}
```

### Generate Secure JWT Key

Generate a cryptographically secure JWT key:

<CodeGroup>
```powershell PowerShell
[Convert]::ToBase64String((1..64 | ForEach-Object { Get-Random -Minimum 0 -Maximum 256 }))
```

```bash Linux/macOS
openssl rand -base64 64
```

```csharp C# (In Code)
var key = Convert.ToBase64String(RandomNumberGenerator.GetBytes(64));
```
</CodeGroup>

### Set JWT Key via Environment Variable

<CodeGroup>
```bash Linux/macOS
export Jwt__Key="your-generated-secure-key-here"
```

```powershell Windows PowerShell
$env:Jwt__Key="your-generated-secure-key-here"
```
</CodeGroup>

<Warning>
  **JWT Key Security**:
  - Minimum 32 characters (64+ recommended)
  - Use cryptographically random generation
  - Never commit to source control
  - Rotate periodically
  - Store in secure vault (Azure Key Vault, AWS Secrets Manager)
</Warning>

## Logging Configuration

### Production Logging Levels

Configure appropriate logging for production:

```json appsettings.Production.json
{
  "Logging": {
    "LogLevel": {
      "Default": "Warning",
      "Microsoft": "Warning",
      "Microsoft.AspNetCore": "Warning",
      "Microsoft.EntityFrameworkCore": "Error",
      "Vantix": "Information"
    }
  }
}
```

### External Logging Providers

Configure third-party logging providers:

#### Application Insights (Azure)

```json
{
  "ApplicationInsights": {
    "ConnectionString": "InstrumentationKey=your-key-here"
  }
}
```

```csharp Program.cs
builder.Services.AddApplicationInsightsTelemetry();
```

#### Serilog

Install package:
```bash
dotnet add package Serilog.AspNetCore
```

Configure:
```csharp Program.cs
using Serilog;

Log.Logger = new LoggerConfiguration()
    .ReadFrom.Configuration(builder.Configuration)
    .CreateLogger();

builder.Host.UseSerilog();
```

## CORS Configuration

### Configure Allowed Origins

For production, specify exact allowed origins:

```json appsettings.Production.json
{
  "Cors": {
    "AllowedOrigins": [
      "https://yourdomain.com",
      "https://www.yourdomain.com",
      "https://app.yourdomain.com"
    ]
  }
}
```

Implement in `Program.cs`:

```csharp Program.cs
var allowedOrigins = builder.Configuration.GetSection("Cors:AllowedOrigins").Get<string[]>();

builder.Services.AddCors(options =>
{
    options.AddDefaultPolicy(policy =>
    {
        policy.WithOrigins(allowedOrigins)
              .AllowAnyMethod()
              .AllowAnyHeader()
              .AllowCredentials();
    });
});

// Later in the pipeline
app.UseCors();
```

<Warning>
  **CORS Security**:
  - Never use `AllowAnyOrigin()` in production
  - Always specify exact domains
  - Use HTTPS for all origins
  - Be cautious with `AllowCredentials()`
</Warning>

## Environment-Specific Settings

### Set Application Environment

The application environment determines which configuration file is loaded:

<CodeGroup>
```bash Linux/macOS
export ASPNETCORE_ENVIRONMENT="Production"
```

```powershell Windows PowerShell
$env:ASPNETCORE_ENVIRONMENT="Production"
```

```xml IIS (web.config)
<environmentVariables>
  <environmentVariable name="ASPNETCORE_ENVIRONMENT" value="Production" />
</environmentVariables>
```
</CodeGroup>

Valid values:
- `Development` - Loads `appsettings.Development.json`
- `Staging` - Loads `appsettings.Staging.json`
- `Production` - Loads `appsettings.Production.json`

## HTTPS Configuration

### Configure Kestrel for HTTPS

```json appsettings.json
{
  "Kestrel": {
    "Endpoints": {
      "Http": {
        "Url": "http://localhost:5000"
      },
      "Https": {
        "Url": "https://localhost:5001",
        "Certificate": {
          "Path": "/path/to/certificate.pfx",
          "Password": "certificate-password"
        }
      }
    }
  }
}
```

<Note>
  When using a reverse proxy (IIS, Nginx), HTTPS is typically handled by the proxy, and Kestrel can listen on HTTP only.
</Note>

## Health Checks

Add health check endpoints for monitoring:

```csharp Program.cs
builder.Services.AddHealthChecks()
    .AddSqlServer(
        builder.Configuration.GetConnectionString("ventixDb"),
        name: "database"
    );

app.MapHealthChecks("/health");
```

## Configuration Checklist

Before deploying to production:

- [ ] `ASPNETCORE_ENVIRONMENT` set to `Production`
- [ ] Connection string configured via environment variable
- [ ] JWT secret key generated and stored securely
- [ ] CORS configured with specific allowed origins
- [ ] Logging configured for production (Warning/Error levels)
- [ ] HTTPS certificate configured
- [ ] `AllowedHosts` configured with production domains
- [ ] Sensitive data removed from `appsettings.json`
- [ ] Health checks configured
- [ ] External logging provider configured (optional)
