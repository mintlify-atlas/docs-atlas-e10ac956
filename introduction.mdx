## Key Features

<Info>
Vantix implements Clean Architecture principles, ensuring separation of concerns, maintainability, and testability across all layers of the application.
</Info>

### Customer Management
- Complete customer lifecycle management
- Document identity validation with unique constraints
- Email and phone contact tracking
- Activity status monitoring
- Automatic registration timestamps

### Modern Technology Stack
- **ASP.NET Core 10**: Latest .NET framework with enhanced performance
- **Entity Framework Core 10**: Powerful ORM with SQL Server support
- **OpenAPI/Swagger**: Auto-generated API documentation
- **Clean Architecture**: Organized in distinct layers for maintainability

### Database-First Approach
Vantix leverages Entity Framework Core's scaffolding capabilities to generate domain models directly from your existing database schema, ensuring perfect synchronization between your database and application code.

## Architecture Overview

The Vantix system is organized into four core projects following Clean Architecture principles:

<Steps>
  <Step title="Vantix.Domain">
    Contains enterprise business rules and entities. This layer has no dependencies on other layers and defines the core domain models.
    
    ```bash
    Vantix.Domain/
    └── Entities/
        └── Models/
            └── Clientes.cs
    ```
  </Step>
  
  <Step title="Vantix.Application">
    Contains application business rules, use cases, and interfaces. This layer depends only on the Domain layer.
    
    ```bash
    Vantix.Application/
    └── [Use Cases & Services]
    ```
  </Step>
  
  <Step title="Vantix.Infrastructure">
    Implements external concerns like database access, file storage, and external APIs. Contains the DbContext and data persistence logic.
    
    ```bash
    Vantix.Infrastructure/
    └── Persistence/
        └── VantixDbContext.cs
    ```
  </Step>
  
  <Step title="Vantix.Api">
    The presentation layer that exposes HTTP endpoints. Contains controllers, middleware, and API configuration.
    
    ```bash
    Vantix.Api/
    ├── Controllers/
    ├── Program.cs
    └── appsettings.json
    ```
  </Step>
</Steps>

## Domain Model Example

Here's an example of a domain entity in Vantix:

```csharp title="Vantix.Domain/Entities/Models/Clientes.cs"
namespace Vantix.Domain.Entities.Models;

public partial class Clientes
{
    public int ClienteId { get; set; }
    public string Nombre { get; set; } = null!;
    public string Apellido { get; set; } = null!;
    public string DocumentoIdentidad { get; set; } = null!;
    public string? Email { get; set; }
    public string? Telefono { get; set; }
    public DateTime? FechaRegistro { get; set; }
    public bool? Activo { get; set; }
}
```
