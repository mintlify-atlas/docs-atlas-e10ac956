## Welcome to the Vantix API

The Vantix Sales System provides a RESTful API built with ASP.NET Core that allows you to manage clients, products, sales, and inventory programmatically.

## Base URL

All API requests should be made to:

```
https://your-domain.com/api
```

## OpenAPI Documentation

Vantix uses OpenAPI (formerly Swagger) to provide interactive API documentation. The OpenAPI specification endpoint is available in development mode.

### Accessing OpenAPI

The OpenAPI endpoint is configured in the application and available at:

```
GET /openapi/v1.json
```

This endpoint provides the complete OpenAPI specification in JSON format, which can be used with tools like Swagger UI, Postman, or other API clients.

### Development Mode

In development mode, the OpenAPI endpoint is automatically mapped and available for use. This is configured in `Program.cs`:

```csharp
if (app.Environment.IsDevelopment())
{
    app.MapOpenApi();
}
```

## API Endpoints

The API is organized around the following resources:

- **Clients** - Manage customer information
- **Products** - Handle product catalog and inventory
- **Sales** - Process and track sales transactions
- **Inventory** - Monitor stock levels and movements

## Getting Started

1. Review the [Authentication](/api/authentication) guide to understand how to authenticate your requests
2. Check the [Error Handling](/api/error-handling) documentation to understand response codes
3. Explore specific endpoint documentation to start making API calls

## SDK and Tools

You can interact with the Vantix API using:

- **HTTP Clients** - Any HTTP client library in your preferred language
- **OpenAPI Tools** - Generate client SDKs from the OpenAPI specification
- **Postman** - Import the OpenAPI spec for easy testing
- **cURL** - Command-line tool for quick API testing

## Rate Limiting

Currently, the API does not enforce rate limiting, but this may be implemented in future versions. Best practices suggest implementing request throttling in production environments.

## Versioning

The current API version is v1. Future versions will be indicated in the endpoint path or headers as needed.
