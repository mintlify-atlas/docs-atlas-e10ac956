## Overview

The Clients endpoint allows you to manage customer information in the Vantix Sales System. You can create, read, update, and delete client records programmatically.

## Base Endpoint

```
/api/clientes
```

## Client Resource

The Client (Clientes) resource represents customer information in the system.

### Client Schema

<ResponseField name="clienteId" type="integer" required>
  Unique identifier for the client (auto-generated)
</ResponseField>

<ResponseField name="nombre" type="string" required>
  Client's first name
</ResponseField>

<ResponseField name="apellido" type="string" required>
  Client's last name
</ResponseField>

<ResponseField name="documentoIdentidad" type="string" required>
  Client's identity document number (e.g., DNI, passport)
</ResponseField>

<ResponseField name="email" type="string">
  Client's email address
</ResponseField>

<ResponseField name="telefono" type="string">
  Client's phone number
</ResponseField>

<ResponseField name="fechaRegistro" type="datetime">
  Date and time when the client was registered
</ResponseField>

<ResponseField name="activo" type="boolean">
  Whether the client account is active
</ResponseField>

### Example Client Object

```json
{
  "clienteId": 1,
  "nombre": "Juan",
  "apellido": "Pérez",
  "documentoIdentidad": "12345678",
  "email": "juan.perez@example.com",
  "telefono": "+51 987654321",
  "fechaRegistro": "2024-01-15T10:30:00Z",
  "activo": true
}
```

## Endpoints

### List All Clients

Retrieve a list of all clients in the system.

<CodeGroup>
```bash cURL
curl -X GET https://your-domain.com/api/clientes \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/clientes', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  }
});

const clients = await response.json();
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://your-domain.com/api/clientes',

clients = response.json()
```

### Get Client by ID

<ParamField path="id" type="integer" required>

</ParamField>

<CodeGroup>
```bash cURL

  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

```javascript JavaScript
const clientId = 1;
const response = await fetch(`https://your-domain.com/api/clientes/${clientId}`, {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
```

### Create New Client

Create a new client record.

<ParamField body="nombre" type="string" required>
  Client's first name
</ParamField>

<ParamField body="apellido" type="string" required>
  Client's last name
</ParamField>

<ParamField body="documentoIdentidad" type="string" required>
  Client's identity document number
</ParamField>

<ParamField body="email" type="string">
  Client's email address (optional but recommended)
</ParamField>

<ParamField body="telefono" type="string">
  Client's phone number (optional)
</ParamField>

<ParamField body="activo" type="boolean">
  Whether the client account is active (defaults to true)
</ParamField>

<CodeGroup>
```bash cURL
curl -X POST https://your-domain.com/api/clientes \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "nombre": "Carlos",
    "apellido": "Rodríguez",
    "documentoIdentidad": "11223344",
    "email": "carlos.rodriguez@example.com",
    "telefono": "+51 998877665",
    "activo": true
  }'
```

```javascript JavaScript
const newClient = {
  nombre: "Carlos",
  apellido: "Rodríguez",
  documentoIdentidad: "11223344",
  email: "carlos.rodriguez@example.com",
  telefono: "+51 998877665",
  activo: true
};

const response = await fetch('https://your-domain.com/api/clientes', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(newClient)
});

const createdClient = await response.json();
```

```python Python
import requests
import json

new_client = {
    "nombre": "Carlos",
    "apellido": "Rodríguez",
    "documentoIdentidad": "11223344",
    "email": "carlos.rodriguez@example.com",
    "telefono": "+51 998877665",
    "activo": True
}

headers = {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Content-Type': 'application/json'
}

response = requests.post(
    'https://your-domain.com/api/clientes',
    headers=headers,
    data=json.dumps(new_client)
)

created_client = response.json()
```
</CodeGroup>

### Delete Client

Delete a client record from the system.

<ParamField path="id" type="integer" required>
  The unique identifier of the client to delete
</ParamField>

<Warning>
  Deleting a client is permanent and cannot be undone. Consider setting the `activo` field to `false` instead to deactivate the client.
</Warning>

<CodeGroup>
```bash cURL
curl -X DELETE https://your-domain.com/api/clientes/1 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

```javascript JavaScript
const clientId = 1;
const response = await fetch(`https://your-domain.com/api/clientes/${clientId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
  }
});
```

```python Python
import requests

client_id = 1
headers = {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN'
}

response = requests.delete(
    f'https://your-domain.com/api/clientes/{client_id}',
    headers=headers
)
```
</CodeGroup>

#### Response

<ResponseExample>
```json 204 No Content
// No response body
```

```json 404 Not Found
{

  "title": "Not Found",
  "status": 404,
  "detail": "Client with ID 1 was not found."
}
```
</ResponseExample>

## Field Validation Rules

<Expandable title="Validation Requirements">
  <ResponseField name="nombre">
    - Required field
    - Must be a non-empty string
    - Maximum length may apply (check server configuration)
  </ResponseField>
  
  <ResponseField name="apellido">
    - Required field
    - Must be a non-empty string
    - Maximum length may apply (check server configuration)
  </ResponseField>
  
  <ResponseField name="documentoIdentidad">
    - Required field
    - Must be a non-empty string
    - Should be unique in the system
    - Format depends on document type (DNI, passport, etc.)
  </ResponseField>
  
  <ResponseField name="email">
    - Optional field
    - Must be a valid email format when provided
    - Example: user@example.com
  </ResponseField>
  
  <ResponseField name="telefono">
    - Optional field
    - Should include country code for international numbers
    - Example: +51 987654321
  </ResponseField>
  
  <ResponseField name="activo">
    - Optional field (defaults to true)
    - Must be a boolean value (true/false)
  </ResponseField>
</Expandable>

## Best Practices

<Tip>
  **Data Quality**: Always provide email and phone information when available. This improves customer communication and record keeping.
</Tip>

### Recommended Workflow

1. **Validate Input**: Check data quality before sending to the API
2. **Handle Errors**: Implement proper error handling for validation failures
3. **Use Filters**: When listing clients, consider implementing pagination for large datasets
4. **Soft Delete**: Use the `activo` field to deactivate clients instead of deleting them
5. **Unique Documents**: Ensure document IDs are unique before creating new clients
