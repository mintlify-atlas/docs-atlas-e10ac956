## Quick Start Guide

This guide will help you get the Vantix Sales System running on your local machine in just a few minutes.

<Info>
For a more comprehensive installation guide with production deployment options, see the [Installation Guide](/installation).
</Info>

## Prerequisites

Before you begin, make sure you have the following installed:

<CardGroup cols={2}>
  <Card title=".NET 10 SDK" icon="microsoft">
    Download from [dot.net](https://dot.net)
  </Card>
  <Card title="SQL Server" icon="database">
    Express, Developer, or full edition
  </Card>
</CardGroup>

Verify your installation:

```bash
dotnet --version
# Should output: 10.x.x
```

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the Vantix repository to your local machine:
    
    ```bash
    git clone https://github.com/GaRIEN/proyecto_ventas_Angular_.net.git
    cd proyecto_ventas_Angular_.net
    ```
  </Step>
  
  <Step title="Restore Dependencies">
    Restore all NuGet packages:
    
    ```bash
    dotnet restore
    ```
    
    This installs:
    - Entity Framework Core 10.0.3
    - SQL Server provider
    - ASP.NET Core OpenAPI
  </Step>
  
  <Step title="Configure Database Connection">
    Update the connection string in `Vantix.Api/appsettings.Development.json`:
    
    ```json title="appsettings.Development.json"
    {
      "ConnectionStrings": {
        "ventixDb": "Server=localhost;Database=VantixV1;Trusted_Connection=True;TrustServerCertificate=True"
      }
    }
    ```
    
    <Tip>
    For SQL Server Express, use: `Server=.\\SQLEXPRESS;Database=VantixV1;...`
    </Tip>
  </Step>
  
  <Step title="Create the Database">
    Create a new database named `VantixV1`:
    
    **Using SQL Server Management Studio:**
    - Connect to your server
    - Right-click "Databases" → "New Database"
    - Name it `VantixV1`
    
    **Using T-SQL:**
    ```sql
    CREATE DATABASE VantixV1;
    GO
    ```
  </Step>
  
  <Step title="Create Database Schema">
    Run this SQL script to create the Clientes table:
    
    ```sql title="Create Clientes Table"
    USE VantixV1;
    GO
    
    CREATE TABLE Clientes (
        ClienteId INT PRIMARY KEY IDENTITY(1,1),
        Nombre NVARCHAR(100) NOT NULL,
        Apellido NVARCHAR(100) NOT NULL,
        DocumentoIdentidad NVARCHAR(20) NOT NULL,
        Email NVARCHAR(150),
        Telefono NVARCHAR(20),
        FechaRegistro DATETIME DEFAULT GETDATE(),
        Activo BIT DEFAULT 1,
        CONSTRAINT UQ_DocumentoIdentidad UNIQUE (DocumentoIdentidad)
    );
    GO
    ```
  </Step>
  
  <Step title="Generate Entity Models">
    Run the scaffold script to generate entity classes from the database:
    
    **Windows:**
    ```bash
    scaffold.bat
    ```
    
    **Linux/Mac:**
    Create `scaffold.sh` with this content:
    ```bash title="scaffold.sh"
    #!/bin/bash
    echo "🚀 Generating models in Domain and Infrastructure..."
    
    dotnet ef dbcontext scaffold \
      "Server=localhost;Database=VantixV1;Trusted_Connection=True;TrustServerCertificate=True" \
      Microsoft.EntityFrameworkCore.SqlServer \
      --project Vantix.Infrastructure \
      --output-dir ../Vantix.Domain/Entities/Models \
      --context-dir Persistence \
      --context VantixDbContext \
      --force \
      --no-pluralize \
      --namespace Vantix.Domain.Entities.Models \
      --context-namespace Vantix.Infrastructure.Persistence
    
    echo "✅ Check your folders now!"
    ```
    
    Then run:
    ```bash
    chmod +x scaffold.sh
    ./scaffold.sh
    ```
    
    <Note>
    The scaffold command generates:
    - Entity classes in `Vantix.Domain/Entities/Models/`
    - DbContext in `Vantix.Infrastructure/Persistence/VantixDbContext.cs`
    </Note>
  </Step>
  
  <Step title="Build the Solution">
    Build all projects to ensure everything compiles:
    
    ```bash
    dotnet build
    ```
    
    You should see:
    ```
    Build succeeded.
        0 Warning(s)
        0 Error(s)
    ```
  </Step>
  
  <Step title="Run the API">
    Start the Vantix API server:
    
    ```bash
    cd Vantix.Api
    dotnet run
    ```
    
    The API will start on:
    - HTTPS: `https://localhost:5001`
    - HTTP: `http://localhost:5000`
    
    <Tip>
    Use `dotnet watch run` for hot reload during development
    </Tip>
  </Step>
  
  <Step title="Test the API">
    Access the OpenAPI documentation:
    
    Open your browser to: `https://localhost:5001/openapi/v1.json`
    
    Or test with curl:
    ```bash
    curl https://localhost:5001/openapi/v1.json
    ```
    
    You should see the OpenAPI specification JSON.
  </Step>
</Steps>


## Common Issues

<AccordionGroup>
  <Accordion title="Connection refused to SQL Server">
    **Solution:** Ensure SQL Server is running and accepting connections.
    
    Check the service status:
    ```bash
    # Windows
    sc query MSSQLSERVER
    
    # Linux
    systemctl status mssql-server
    ```
  </Accordion>
  
  <Accordion title="Build failed: SDK not found">
    **Solution:** Install .NET 10 SDK from https://dot.net
    
    Verify:
    ```bash
    dotnet --list-sdks
    ```
  </Accordion>
  
  <Accordion title="Scaffold fails: dotnet-ef not found">
    **Solution:** Install EF Core tools:
    ```bash
    dotnet tool install --global dotnet-ef
    ```
  </Accordion>
  
  <Accordion title="Database login failed">
    **Solution:** Check your connection string authentication method.
    
    For SQL Authentication:
    ```json
    "ConnectionStrings": {
      "ventixDb": "Server=localhost;Database=VantixV1;User Id=sa;Password=YourPassword;TrustServerCertificate=True"
    }
    ```
  </Accordion>
</AccordionGroup>
